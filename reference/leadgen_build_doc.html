01 · The Brief All

A lead-generation pipeline the sales team operates without engineering support, on infrastructure we already pay for.

Everything below is open-source. No SaaS subscriptions in Phase 1. The pipeline scrapes Google Maps for prospects in our target verticals (fuel retail, abaya / fashion, auto parts, real estate, hospitality across Kerala & GCC), enriches and qualifies each lead with AI scoring, and surfaces them to the sales team in an Airtable-style interface for manual outreach via WhatsApp. Phase 2 adds automated WhatsApp messaging via a Business API provider when volume justifies it.

▸ Outcome

50–200 qualified leads in the sales pipeline every week.

Pre-scored, pre-deduped, with phone + name + business context ready for outreach. No CSV cleanup, no manual research.

▸ Cost

~ ₹500–1500 / month at Phase 1 volumes.

Only cost is Claude API calls for AI scoring (~₹0.30 per lead). Hosting uses existing VPS. No CRM or scraper subscriptions.

▸ Timeline

10 working days from sign-off to first live lead.

Zo deploys in 2–3 days. Sales team onboarding 1 day. First scrape and outreach by end of week 2.

What "good" looks like at 60 days: Aslam and the sales team run 3 new search queries each week without asking Zo for help. Every Monday morning a fresh batch of 100+ AI-scored leads appears in the pipeline. Reply rate on cold WhatsApp outreach holds above 8%. At least one signed engagement directly attributed to a lead from this system within the first 60 days.

02 · Architecture IbrahimZo

Five components. One VPS. One subdomain. Everything dockerised.

The whole stack runs as five Docker containers on the existing Adaline VPS. Sales team accesses it at leads.myadaline.com behind a login. Data lives in a single PostgreSQL database. The scraper, the workflow engine, and the sales UI are wired together through that database — no fragile API chains.

▸ The five components

gosom/google-maps-scraper
scraper

Extracts business listings from Google Maps. Built-in Web UI lets the sales team run queries without touching code. Writes results directly to PostgreSQL via the LeadsDB feature.

License MIT · open source | Stars 10K+ | Updated May 2026
PostgreSQL
database

Single source of truth for every lead. Stores raw scraper output, AI scoring, sales team status updates, and outreach history. Backed up nightly.

License PostgreSQL (BSD-style) | Storage ~50MB per 10K leads
n8n
workflow

Free, self-hosted alternative to Zapier. Conducts the pipeline — picks up new leads from PostgreSQL, calls Claude for scoring, writes results back. Has a visual workflow editor.

License Sustainable Use (free for internal use) | Workflows visual builder
NocoDB
sales UI

Open-source Airtable. The sales team's primary working surface. Connects to PostgreSQL and gives a spreadsheet-style view with filters, status columns, assignees, and exports.

License AGPL-3.0 | Stars 50K+ | Auth per-user accounts
Caddy
proxy

Reverse proxy with automatic HTTPS via Let's Encrypt. Wraps the whole stack behind one subdomain and a login screen. Configure once, forget about it.

License Apache-2.0 | HTTPS auto-provisioned
Claude Haiku API
AI · external

The only paid component. ~₹0.30 per lead scored. Returns score 1–10, qualified Y/N, and a short reasoning string. Replaceable with any LLM API later if needed.

Cost ~₹500–1500/month | Provider Anthropic

04 · Build Phases All

Two phases. Ship Phase 1. Earn Phase 2.

Phase 1 is the entire stack except automated outreach. It runs at near-zero recurring cost and gives the sales team a working pipeline. Phase 2 layers WhatsApp Business API automation on top — but only after Phase 1 produces enough qualified leads to justify the ~₹3,500/month BSP subscription.

▸ Phase 01 BUILD NOW

Free stack, manual outreach.

Everything dockerised on the existing VPS. Sales team runs queries, works the AI-scored pipeline, reaches out manually via their own WhatsApp Web. This is shippable and useful from day one.

  • gosom scraper deployed and accessible
  • PostgreSQL + nightly backups
  • n8n workflow: dedupe + enrich + score
  • Claude Haiku API integration for scoring
  • NocoDB sales pipeline view
  • Caddy + HTTPS + basic auth
  • 5 search-query templates pre-loaded for active verticals
  • Sales team SOP doc + 1 training session
Monthly cost~ ₹500–1500
▸ Phase 02 LATER · TRIGGERED

Automated outreach layer.

Add WhatsApp Business API via AiSensy or Interakt. n8n triggers personalised first messages to AI-qualified leads. Replies route back to NocoDB where the assigned sales rep takes over. Only build this once Phase 1 is producing leads worth automating.

  • AiSensy or Interakt subscription + WhatsApp business number verification
  • n8n: trigger workflow for "qualified + not yet contacted" leads
  • Per-vertical message templates with personalisation tokens
  • Reply webhook into NocoDB — thread continues with human
  • Rate-limiting and quiet-hours rules
  • Per-lead cost tracking (utility vs marketing messages)
Monthly cost~ ₹3,500–6,000
▸ Phase 2 trigger criteria — when to build
Decision rule

Phase 1 has produced 100+ qualified leads in active pipeline AND at least one closed deal directly attributed to this system AND sales team request ("manual outreach is the bottleneck, we can't keep up"). All three. Don't automate before validation — message templates that haven't been pressure-tested against real replies waste money fast.

init.sqlrun after stack is up
-- The single table that holds everything.
-- gosom writes rows on scrape. n8n updates score columns.
-- NocoDB reads and lets sales team update status columns.

CREATE TABLE leads (
  id            SERIAL PRIMARY KEY,
  scraped_at    TIMESTAMPTZ DEFAULT NOW(),

  -- from gosom scraper
  name          TEXT NOT NULL,
  category      TEXT,
  address       TEXT,
  city          TEXT,
  country       TEXT,
  phone         TEXT,
  email         TEXT,
  website       TEXT,
  rating        DECIMAL(2,1),
  review_count  INTEGER,
  query_used    TEXT,

  -- written by n8n + claude
  score         DECIMAL(3,1),       -- 0.0 to 10.0
  qualified     BOOLEAN,
  ai_reason     TEXT,              -- short rationale
  scored_at     TIMESTAMPTZ,

  -- managed by sales team in NocoDB
  status        TEXT DEFAULT 'new',
  assigned_to   TEXT,
  last_contact  TIMESTAMPTZ,
  next_action   TEXT,
  notes         TEXT,
  outcome       TEXT,              -- closed-won / lost / etc

  -- dedup key
  UNIQUE(phone, name)
);

CREATE INDEX idx_status ON leads(status);
CREATE INDEX idx_assigned ON leads(assigned_to);
CREATE INDEX idx_score ON leads(score DESC);

Why not a NocoDB formula: NocoDB's formula language doesn't have proper URL encoding, and phone numbers come in inconsistent formats (with/without +, with/without country code, with spaces, with dashes). A formula that handles all the edge cases gets unmaintainable fast. Cleaner: n8n generates the URL once during scoring and writes it to a whatsapp_url column. NocoDB just shows it as a clickable button.

// Normalise phone to international format, strip non-digits
function normalisePhone(raw, country) {
  const digits = raw.replace(/[^0-9]/g, '');
  // already has country code?
  if (digits.startsWith('971') || digits.startsWith('966') ||
      digits.startsWith('974') || digits.startsWith('973') ||
      digits.startsWith('968') || digits.startsWith('965') ||
      digits.startsWith('91')) return digits;
  // add based on country field
  const codes = {UAE:'971', KSA:'966', Qatar:'974',
    Bahrain:'973', Oman:'968', Kuwait:'965', India:'91'};
  return (codes[country] || '91') + digits;
}

// Build pre-filled WhatsApp message based on vertical
function buildMessage(lead) {
  const templates = {
    abaya: `Hi! Saw ${lead.name} — really like the brand aesthetic. We're Adaline, a brand and content studio in Calicut working with abaya labels across the GCC. Open to a quick conversation?`,
    autoparts_b2b: `Hi! Came across ${lead.name} — looks like a serious operation in ${lead.city}. We're Adaline, building digital systems for auto-parts distributors. Currently working with Roca Group. Open to a peer call?`,
    fuel: `Hi! Saw ${lead.name} on Google. We're Adaline, currently running a 36-month brand programme with Roca Fuels (MRPL dealer in Calicut). Open to a quick walk-through of what's working?`
  };
  return templates[lead.vertical_tag] || templates.abaya;
}

const phone = normalisePhone($json.phone, $json.country);
const message = encodeURIComponent(buildMessage($json));
$json.whatsapp_url = `https://wa.me/${phone}?text=${message}`;
return $json;

▸ WhatsApp outreach rules — the discipline

Send between 10 AM and 6 PM local time of the lead
Etiquette

Calicut leads: 10 AM–6 PM IST. Gulf leads: shift to 10 AM–6 PM GST (UTC+4). Never on Friday in Gulf countries. Never after 8 PM anywhere. Out-of-hours messages mark you as spam in the lead's head before they've read a word.

Personalise the first sentence — every time
Discipline

Generic template = ignored. The pre-filled message has a placeholder for one specific detail: a product, a location, a review highlight, an Instagram post. Take 30 seconds, fill it in, then send. This single discipline is the difference between 2% reply rate and 12% reply rate.

Never message the same lead twice from different sales reps
Critical

The "Assigned to" column exists for this. Once you claim a lead, it's yours. Another team member messaging the same business looks unprofessional and burns the relationship. Check "Assigned to" before reaching out.

Track everything in NocoDB, not in your head
Habit

Notes after every interaction. Next-action date for every "contacted" or "replied" lead. If it's not in NocoDB, it didn't happen — and Ibrahim's Friday report won't reflect it.

10 · Risks & Honest Caveats All

What can go wrong, and how we'd handle it.

Listed in order of likelihood. The whole stack has been used in production by other teams — none of these are theoretical, all have been seen.

Google rate-limits the VPS IP after high-volume scraping
Likely · Phase 1

If we push past ~500 queries per day from a single IP, Google starts serving captchas or empty results. Phase 1 stays well below this if queries are spread across the week.

Mitigation — Cap scraping to 200 results per query, run no more than 3 large queries per day. If we hit limits, pause for 24 hours. Phase 2 includes residential proxies (~₹4,000/month) when volume justifies.
Email coverage is patchy (30–50% of leads have an extracted email)
Certain · accept it

Most small businesses don't list emails publicly. gosom visits the website and greps for any email address it finds — info@, contact@, etc. — but plenty of businesses just use a contact form or only WhatsApp.

Mitigation — Phone numbers are near-100% covered. WhatsApp is our primary channel anyway. Treat email as a bonus, not a target.
Sales team doesn't maintain NocoDB hygiene
Likely · culture

Without discipline, the pipeline becomes a graveyard of "new" leads nobody touched and "contacted" leads with no follow-up date. Within 4 weeks the system loses trust and gets abandoned.

Mitigation — Aslam enforces the daily flow. Friday weekly numbers go to Ibrahim regardless of whether they're impressive. Ibrahim chases gaps in the Monday standup. Visible accountability.
Claude API key is leaked or abused
Low · severe if it happens

Anyone with the key can spend on our account until we notice.

Mitigation — Key lives only in the .env file on the VPS, never committed to git, never shared in chat. Set spend cap on the Anthropic account at ₹3,000/month — anything beyond that gets paused automatically. Rotate the key quarterly.
A scraped lead complains about cold outreach
Possible · per market

India: DPDP Act 2023 requires consent for marketing communications. WhatsApp business outreach to a publicly listed number for a B2B inquiry is on the line — defensible but not bulletproof. GCC: similar story.

Mitigation — Frame every first message as a genuine B2B inquiry, not a marketing blast. Honour any "remove me" request immediately, set status to "lost · opt-out", never message that number again. Document opt-outs in NocoDB. Don't blast — qualify and target.
Email deliverability tanks if we ever shift channels
Phase 2 concern

If anyone tries to use this stack to push cold emails through our domain at volume, we'll get blacklisted within a week and lose all legitimate email reachability.

Mitigation — This stack is WhatsApp-first by design. If email outreach is ever added, it goes through a dedicated outreach domain (e.g., reachout-adaline.com) with proper SPF/DKIM/DMARC, warmup, and per-domain sending caps. Not adaline.com.
Stack itself is stable
Low risk · maintained

All five components are actively maintained, well-funded or community-supported, and have been in production at thousands of other small teams. Outages will be self-inflicted (config errors, expired API keys) rather than vendor-side.

Confidence — gosom updated May 2026. NocoDB 50K+ stars, weekly releases. n8n VC-backed, enterprise customers. PostgreSQL is PostgreSQL. Caddy maintained by a team. Zo is supported.
